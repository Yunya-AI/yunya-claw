## 所以我做了Yunya Claw

它本质上就是一个OpenClaw的桌面客户端，基于Electron封装，内置Node.js运行环境。

跟其他客户端一样，双击安装，打开就能用。

但它多做了一件事：**让你可以把它变成完全属于你自己的应用。**

## 改名字，改头像，从里到外都是你的

这是Yunya Claw跟其他OpenClaw客户端最大的区别。

打开设置页面，你会看到"应用外观"这一栏：
- **应用名称**：改成你想要的任何名字，窗口标题和任务栏立刻更新
- **应用图标**：上传一张PNG或JPG，马上生效

比如截图里，我把它改成了"MyClaw"，配了一个自己喜欢的头像。

你也可以改成你公司的名字加上公司logo，直接当企业内部AI助手用。

改完之后，**从窗口标题到侧边栏，看不出任何Yunya Claw的痕迹**。

这不是换皮，是真正的白标能力。

## 原生封装，不是套壳浏览器

说一下技术层面的事。

市面上有些OpenClaw客户端本质上是开个浏览器窗口指向本地服务。Yunya Claw不一样，它是用Electron原生封装的：

- **内置Node.js 22运行环境**，不依赖系统安装的任何东西
- **原生窗口管理**，不是iframe嵌套
- **IPC通信**直接调用底层API，头像、名称这些自定义能力就是通过原生接口实现的

这也是为什么"改名字改头像"这件事在其他客户端上做不到——因为它们的架构不支持。

## OpenClaw的能力，一个不少

作为OpenClaw客户端，该有的功能都有：

### 人设系统

OpenClaw的6个核心设定文件，在界面上直接编辑：

- **系统提示词（AGENTS.md）**：操作指令、记忆、行为规则
- **人设（SOUL.md）**：角色设定、语气、边界
- **身份（IDENTITY.md）**：名称、风格
- **用户信息（USER.md）**：你的偏好、称呼
- **工具指南（TOOLS.md）**：工具使用规范
- **心跳检查（HEARTBEAT.md）**：定期运行的检查清单

不用去翻文件夹找Markdown文件了，界面上点开就能改。

## 跟其他OpenClaw客户端比，区别在哪？

说实话，核心的AI能力大家都一样，毕竟底层都是OpenClaw。

区别在上层体验：

| | 其他客户端 | Yunya Claw |
|---|---|---|
| 一键安装 | 支持 | 支持 |
| 自定义应用名称 | 不支持 | 支持 |
| 自定义应用头像 | 不支持 | 支持 |
| 白标/企业定制 | 不支持 | 支持 |
| 原生Electron封装 | 部分是 | 是 |
| 内置Node.js运行环境 | 部分有 | 有 |
| 社交平台接入UI | 需要配置文件 | 可视化界面 |
| 一键备份恢复 | 不一定有 | 内置 |

简单说：**如果你只是想用OpenClaw，任何客户端都行。如果你想让OpenClaw变成"你自己的应用"，Yunya Claw目前是唯一的选择。**

## 安装

跟其他客户端一样简单：

## 关于数据

所有数据存在本地 `~/.openclaw/`，跟OpenClaw生态完全兼容。

应用内置了一键备份和恢复功能，配置、人设、定时任务都能打包保存。
